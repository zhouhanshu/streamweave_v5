任务说明：
1. **状态与回答**：
   - 在写 <answer> 之前，先写一个简短的 <state> ，总结相关的 Memory 和当前帧。Memory 包含到目前为止观察到的视频历史，当前帧是即时的视频窗口；二者都按时间戳排序。结合二者理解视频，但是要尽可能简短。检查 QA History，以判断现在是否应该回答问题。如果你认为可以基于视频 Memory 或当前帧回答问题，就在 <answer> 中写出答案。如果没有活跃问题，或者你认为问题目前还不能回答，就保持 <answer></answer> 为空。
   - 回答问题时，要遵循问题的格式，问题可能是选择题，那你就回答选项，或者是不能回答时可以选择不能回答的选项（如果有的话）
   - **上下文分析**：仔细检查 "QA History" 日志，并根据问题本身确定适当的证据范围。随着视频推进，一个问题可能需要多次更新。如果问题涉及当前正在发生的动作、状态或空间关系，判断应主要依赖最新帧，历史记忆仅作为背景上下文。如果问题涉及过去的过程、物体位置、状态变化或累计结果，判断应主要依赖 Memory 中的 notes 和 bridges，并由当前帧补充。
   - **静默标准**：如果 QA History 中没有活跃问题，或者当前帧没有为已经回答的问题提供有用更新，则 <answer> 保持为空。
   - **State 非记忆**：<state> 只用于本轮判断，不会写入 Memory，也不会在后续轮次中保留。任何后续可能需要用到的事实、位置、文字、状态变化、物体属性或事件结果，必须通过 <bridge> 或 <note> 保留下来；不要只写在 <state> 中。

2. **Note 锚定策略**：
   - **强制初始化**：如果 Memory 是 <empty/>，第一个观察标签必须是针对 窗口内第一帧的 <note>。
   - **锚定原则**：<note> 用来保存关键视觉证据。如果某一帧承载了无法仅通过 bridge 可靠保存的信息，应创建一个 <note>。
   - **状态变化锚定**：当场景、相机视角、工作区布局、任务阶段、关键物体位置或可见状态发生显著变化，并且该变化可能影响之后的理解时，应创建一个 <note>。
   - **代表性选择**：如果当前窗口中有多帧满足锚定条件，只选择最能代表当前状态、最能支持 QA 或最能保存关键视觉证据的一帧。不要因为细微动作或姿态变化创建多个 note。
   - **转录**：输出 note 时，将 "Current frames" 区块中的时间范围复制到 `t` 属性中，以确保精确锚定。

3. **Bridge 压缩与继承**：
   - **先判断尾部类型**：输出观察标签前，检查 Memory 中最后一个观察标签。如果最后一个标签是 <bridge t="A-B">...</bridge> 且其后没有 note，则视为开放尾部；如果最后一个标签是 note，则不存在开放尾部；如果 Memory 是 <empty/>，先按初始化规则输出窗口第一帧 note。
   - **开放尾部继承（关键）**：若存在开放尾部，第一个输出 bridge 必须继承原 bridge 的起始时间 A，并作为对原 bridge 的替换式更新。不要重复输出旧 bridge，也不要从旧 bridge 的结束时间 B 开始新 bridge。若当前窗口没有新 note，写为 <bridge t="A-C">...</bridge>，C 为当前窗口结束时间；若当前窗口有新 note，写为 <bridge t="A-N">...</bridge>，N 为第一个当前 note 的起始时间，随后输出该 note。
   - **非开放尾部起点**：若不存在开放尾部，新输出从 Memory 最后一个观察标签的结束时间开始；如果该时间早于当前窗口起点，则从当前窗口起点开始。不要覆盖、重复或重写 Memory 中已经闭合的时间段。
   - **间隙唯一性**：在任意两个相邻 notes 之间，或在最后一个 note 与当前窗口结束边界之间，保持恰好一个 bridge。不要把同一时间间隙拆成多个连续 bridges；即使变化很少，也用一个简短 bridge 描述状态延续。
   - **时间对齐**：除开放尾部继承外，所有输出标签必须按时间顺序排列，时间连续、无重叠、无倒退。note 的 t 属性必须原样复制 Current frames 中对应帧的时间范围；bridge 的起止时间连接相邻 note 或当前窗口边界。
   - **内容原则**：bridge 只描述该时间段内可观察到的主要动作、状态变化和事件边界，准确简洁，不补不确定细节。重复动作应保留已完成动作的累计结果；准备中、进行中、被遮挡或不确定的动作不得写成已完成。工具、颜色、朝向、身份、文字和物体属性等细节，只有清晰可见或被 Memory 明确确认时才写出；不确定时保守描述或省略。
