# 补充实验待办

## 核心命题

**图文交错记忆(Anchor + Delta)优于纯文本记忆,因为图片锚点保留原始视觉证据,模型在相似项之间不容易混淆。**

需要从两个角度给出证据:

- 直接证据:在 memory 这一中间层,量化图文记忆相比纯文本记忆的语义保真度。
- 间接证据:在下游 QA 上,图文记忆在"容易混淆"的题目上准确率更高。

## 直接证据:Memory Fidelity Evaluation

设定:

- 输入原视频已观测前缀的 1fps 抽帧。
- 对同一前缀生成多种候选 memory:至少包括纯文本 memory 和 Anchor+Delta 图文交错 memory。
- 用强 VLM judge(例如 Qwen3-VL-32B)基于原始抽帧给候选 memory 打分。
- 所有候选 memory 共享同一视频前缀,禁止使用未来帧。

核心指标:

- `Coverage`:memory 是否覆盖关键对象、动作、状态变化、OCR、颜色、空间关系、数量、身份。
- `Factuality`:memory 中的描述是否被原视频帧支持,惩罚编造和错误属性。
- `Chronology`:memory 是否保持事件和状态变化的正确时间顺序。
- `Token Efficiency`:上述分数除以 memory token 数,衡量单位 token 的信息保真。

产物:JSONL,每行包含 `video_id`、`prefix_end_time`、`memory_type`、`token_count`、各项分数、`missed_evidence`、`unsupported_claims`。

## 间接证据:三个混淆类幻觉评测

三个评测各自针对一类典型混淆,正好对应图片锚点应该能消除的失败模式。

- VideoHallucer:`https://huggingface.co/datasets/bigai-nlco/VideoHallucer`
  - 对象属性 / 关系混淆,问题前提不成立时是否硬答。
- EventHallusion:`https://github.com/Stevetich/EventHallusion`
  - 相似物体混淆,例如 wire/scissors、belt/blue item。
- VidHalluc:`https://vid-halluc.github.io/`
  - 相似事件、时序、场景切换混淆。

需要一个统一转写脚本,把以上三个数据转成 StreamWeave 可跑的 video QA JSONL,并额外标注 `answerable`、`false_premise`、`near_miss_negative`、`requires_unable`,用于后续训练 hard-negative 数据复用。

## 记录要求

每个补充实验至少记录:

- 启动脚本和关键 override。
- 初始化模型、数据路径、输出目录。
- 对比组定义(至少 Anchor+Delta vs 纯文本)。
- 关键指标值和显著性。
- 结论:命题是否被支持,以及下一步动作。
