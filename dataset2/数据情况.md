本文档只记录 `dataset2` 当前要使用的数据、关键清洗规则和必要的追溯信息。当前主要数据如下：

| 用途 | 文件 | 数量 | 状态 |

## 当前 SFT 数据

最终文件：

```text
dataset2/sft_0511_note.jsonl
```

格式：LLaMAFactory ShareGPT，字段为 `messages` 和 `images`；图片路径按 `dataset2` 作为 `media_dir` 解析。

规模：

| 类型 | 行数 |
|---|---:|

| answered，有 answer step | 80,295 |
| **合计** | **182,998** |

来源分布：

| 来源 | 行数 | 占比 |
|---|---:|---:|
| `LLaVA-Video-178K-YouTube` | 127,941 | 69.9% |

| `LLaVA-Video-178K-PerceptionTest` | 2,719 | 1.5% |

生成规则：

- 来源是五份 hard-filtered SFT 蒸馏数据合并后的 `dataset2/sft_0511.jsonl`。
- 从 `347,695` 行中按完整视频/rollout 的最终 memory 统计 note/anchor 占比。
- 保留完整样本 note/anchor 占已看帧数在 `[9%, 15%]` 的样本，得到 `182,998` 行。
- 保持原文件顺序写出。
- 当前协议使用 `<anchor>/<delta>`，不是旧的 `<note>/<bridge>`。

硬过滤规则：

- ShareGPT 结构错误、JSON 错误、缺少 `<answer>`、strict parser 失败则删除。
- `images` 中任一图片不存在则删除；图片按 `--image-root dataset2` 解析。
- 任一 `<delta>` 时间跨度 `> 20s` 则删除。
- 完全重复行删除。
- 输入 memory 为 `<empty/>` 时，第一条 observation 必须是 `<anchor t="0.0-1.0"></anchor>`。

### SFT-new note 占比过滤版

最新候选文件：

```text
dataset2/sft_0511_new.jsonl
```

来源链路：

```text
dataset2/sft_0511.jsonl
  -> 按完整视频/rollout 的最终 memory 统计 note/anchor 占比
  -> 保留 note/anchor 占已看帧数在 [9%, 15%] 的完整样本
  -> 得到 dataset2/sft_0511_note.jsonl
  -> 按空 answer:有 answer = 6:4 降采样
  -> 得到 dataset2/sft_0511_new.jsonl
```

`note` 兼容统计 `<anchor>` 和旧 `<note>`。完整样本按 `dataset/video_id` 分组，取该组最后一个 SFT step 的最终 memory 统计；占比为唯一 note/anchor 时间戳数除以最后 current frame 的结束时间，当前数据为 1fps，因此近似为 note 帧数 / 已看帧数。

| `dataset2/sft_0511_note.jsonl` | 182,998 | 4,127 | 80,295 | 102,703 |

note-filter 统计：

| 指标 | 数值 |
|---|---:|
| 完整样本 note/anchor 加权占比 | 11.82% |
| 单样本平均占比 | 11.95% |
| 单样本 p50 占比 | 11.81% |
| 单样本 p95 占比 | 14.71% |

最终 new 文件规模：

| 类型 | 行数 | 占比 |

字段说明：

| 字段 | 含义 |
|---|---|
| `sample_id` | 展开后的唯一 ID，格式 `{video_id}__{qa_id}` |
| `dataset` | 帧目录所属数据集名，RL 按它找 `dataset2/{dataset}/video/{video_id}` |
| `video_id` | 视频 / 帧目录 ID |
| `video`, `frames_dir` | 相对 `dataset2` 的帧目录参考路径 |

| `frame_id_base` | 帧编号基准，当前为 0 |
| `realtime` | 问题注入时间，当前基本是视频末尾 |
| `question`, `options`, `answer`, `gt` | 单个 QA 的题目、选项、文本答案和标准答案 |
| `pass_count` | SFT 后模型完整 StreamWeave 推理 3 次的答对次数 |
| `last4_pass` | 只看最后 4 帧用 Gemini Flash 答题是否答对，`1` 对、`0` 错 |
| `parent_qa_count` | 所属视频在 `rl_filtered_2.jsonl` 中保留下来的 QA 数 |

## RL 数据生成规则

`rl_0511.jsonl` 的来源链路：

```text
rl_tmp.jsonl
  -> SFT 模型 3 次完整 StreamWeave 推理，写入 pass_count
  -> 按 pass_count 做 QA 难度配比过滤
  -> Gemini Flash 只看最后 4 帧打 last4_pass
  -> last4_pass=0 全保留，last4_pass=1 保留 20%
  -> 展开 qa_list，合并为 dataset2/rl_0511.jsonl
```

关键规则：

- `pass_count=0` 的 QA 删除。
- `pass_count=1/2/3` 分别视为 hard/medium/easy。
- 第一轮按 QA 级 `easy:medium:hard = 2:5:3` 做全局采样。
- 对 YouTube 第一轮结果按视频时长升序保留 1,000 个视频。
- 最后 4 帧测试只用于控制“只靠末尾静态帧可答”的比例。
- `last4_pass=0` 全保留；`last4_pass=1` 固定随机种子 `511` 保留约 20%。
- 展开后一行一个 QA，删除 `qa_list` 和 `qa_count`，新增 `parent_qa_count`。

中间文件：

- `dataset2/NeXTVideo/rl_filtered_2.jsonl`
- `dataset2/LLaVA-Video-178K-PerceptionTest/rl_filtered_2.jsonl`
- `dataset2/LLaVA-Video-178K-YouTube/rl_filtered_2.jsonl`

这些只是 `rl_0511.jsonl` 的来源，不作为当前 RL 训练入口。

## 数据集使用决策

| 数据集 | 当前用途 | 说明 |
|---|---|---|
| `NeXTVideo` | SFT + RL | 偏 temporal / causal VideoQA，适合历史记忆 |
| `LLaVA-Video-178K-PerceptionTest` | SFT + RL | 短视频 3 选 1，作为感知补充 |
| `LLaVA-Video-178K-YouTube` | SFT + RL | 大规模 4 选 1；已用 last4 过滤降低静态题占比 |
| `TimeChat-Online-139K` | SFT | 开放式 QA，只进 SFT，不进当前 RL 自动 MCQ 链路 |
| `CogStream` | 停用 | 当前版本不进入 SFT、RL、自动打分和数据分配 |
| `Streamo-Instruct-465K` | 暂不使用 | 已有可映射子集，但不进入当前两份最终数据 |
