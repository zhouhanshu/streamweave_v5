### CRR 重跑

官方 OVO offline / AURA 路径确认：

- Forward 任务按每个 `test_info[i]` 单独切成 `id_i.mp4`
- `id_i.mp4` 的结尾是 `test_info[i].realtime`
- inference 对这个 chunk 问一次，写入 `test_info[i]["response"]`
- score 直接用 `test_info[i]["response"]` 和 `type/count` 判分

因此已将本仓库 `evaluation/ovo_adapter.py` 中 Forward 样本的 query time 改为统一使用 `test_info.realtime`：

```python
query_timestamp = target_timestamp
```

也就是 `REC / SSR / CRR` 都在各自 chunk 末尾问一次。

旧 CRR 行已从 active 输出中删除：

```text
results.jsonl: 3035 -> 2795，CRR=0
.results_parts: 3035 -> 2795，CRR=0
pending expected = 240
```

## StreamingBench：Base / SFT / RL

时间：2026-05-19

评测链路：

- runner：`scripts/run_streamingbench_8gpu_vllm.sh`
- evaluator：`evaluation/eval_batch.py --benchmark streamingbench`
- adapter：`evaluation/streamingbench_adapter.py`
- prompt：`eval`
- postprocess：`eval_repair`
- sample_fps：`1.0`
- frames_per_step：`5`
- memory.window_seconds：沿用 StreamingBench 配置默认 `120`
- workers：`32`
- GPUs：`0 1 2 3 4 5 6 7`

StreamingBench split：

| Split | Config | Grouped by video | Task filter | Output suffix |
|---|---|---:|---|---|
| real | `configs/eval_streamingbench.yaml` | yes | - | `_real` |
| sqa | `configs/eval_streamingbench.yaml` | no | - | `_sqa` |
| omni | `configs/eval_streamingbench.yaml` | yes | `Misleading Context Understanding,Anomaly Context Understanding` | `_omni_mislead_anomaly` |
| proactive | `configs/eval_streamingbench_proactive.yaml` | no | - | `_proactive` |

链路检查：

- `runtime.resolution` 会在 `backend/openai.py` 编码 image data URL 时作为 `max_side` 生效。
- loader 读数：real = `499` 个 grouped video / `2500` 个结果行；sqa = `250` 行；omni = `100` 个 grouped video / `500` 个结果行；proactive = `235` 行。
- proactive 当前有 `3` 个 source video 缺失，因此跳过 `15` 个样本；这是数据文件现状，不是脚本分辨率或模型配置问题。
