# 数据准备

本文只记录上游数据如何下载/落盘、如何清洗抽帧成 StreamWeave 统一格式。SFT 合成清洗见 `02-SFT数据清洗.md`，RL schema、过滤、难度筛选见 `03-RL数据清洗.md`。

## 统一目标格式

所有上游数据最终都落到 `dataset2/<dataset>/`：

```text
dataset2/<dataset>/
  video/
    <video_id>/
      000000.jpg
      000001.jpg
      ...
      manifest.json
  annotations.jsonl / sft.jsonl / rl_tmp.jsonl / 其他下游标注
```

统一约定：

- 帧目录统一是 `video/<video_id>/`。
- 抽帧统一按 `sample_fps = 1.0`，即 1fps。
- 帧文件从 `000000.jpg` 开始连续编号，`frame_id_base = 0`。
- `manifest.json` 记录 `status`、`frame_count`、`sample_fps`、`frame_id_base`、抽帧来源等信息。
- 标注里必须能通过 `dataset + frames_dir/video` 找到对应帧目录。
- 有效视频长度以下游标注里的 `frame_count` 为准；真实目录里多出来的帧不自动作为训练长度。

## 上游数据来源

当前主线涉及六个上游数据目录：

```text
dataset2/CogStream/
dataset2/Streamo-Instruct-465K/
dataset2/NeXTVideo/
dataset2/LLaVA-Video-178K-YouTube/
dataset2/LLaVA-Video-178K-PerceptionTest/
dataset2/TimeChat-Online-139K/
```

下载/落盘口径：当前代码不是在线下载器，旧处理脚本都假设官方数据包、HF 数据或本地原始视频已经先放到 `/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/` 和 `/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/raw/`。所以数据准备阶段真正要检查的是：上游文件是否已经按脚本要求落盘，能否从这些本地源目录抽帧到 `dataset2/<dataset>/video/`。

这些目录里的 `video/` 是已经抽好的统一帧目录。旧的原始处理脚本和中间文件已经收进 `dataset2/trash/dataset_dirs/<dataset>/tmp/`，用于追溯，不再作为当前下游入口。

## CogStream

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/CogStream/
```

原始数据结构是官方 CogStream 的 train/test 划分：

```text
CogStream/
  Train_dataset/
    VQA_Dataset/*.json
    event_segments/<video_name>/*_segment_N.mp4
  Test_dataset/
    VQA_Dataset/*.json
    event_segments/<video_name>/*_segment_N.mp4
```

处理脚本：

```text
dataset2/trash/dataset_dirs/CogStream/tmp/process_cogstream.py
```

处理逻辑：

1. 读取 `VQA_Dataset/*.json` 里的 QA、`segment_timestamp`、`event_timestamp`、`label`、`COR`、`relevance` 等字段。
2. 只保留可视、非历史依赖、窗口长度合适的 QA；`history_cor`、`history_relevance`、`dialogue_recalling` 等样本被过滤，不进入当前主线。
3. 按原视频的 `event_segments` 顺序抽成 1fps 原始帧缓存。
4. 每个输入窗口生成一个 `video/<video_id>/`，窗口从 0 秒开始，到 `segment_timestamp` 对应的帧结束。
5. 多个 QA 如果使用同一个视频前缀窗口，可以共享同一个帧目录。

当前保留：

```text
dataset2/CogStream/video/
dataset2/CogStream/annotations.jsonl
```

其中 `annotations.jsonl` 是保留的 CogStream 原始转换标注副本。

## Streamo-Instruct-465K

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/Streamo-Instruct-465K/
```

Streamo 的 QA 来自多个源文件，旧脚本默认处理：

```text
ActivityNet.json
LLaVA_Video.json
QVHighlight.json
```

旧脚本默认跳过：

```text
Llava_video_offline.json
```

处理脚本：

```text
dataset2/trash/dataset_dirs/Streamo-Instruct-465K/tmp/filter_streamo_qa.py
dataset2/trash/dataset_dirs/Streamo-Instruct-465K/tmp/process_streamo.py
```

处理逻辑：

1. `filter_streamo_qa.py` 先从 Streamo 原始 JSON 中解析 `question` / `response`。
2. 根据 `ask_time` 和 `answer_time` 判断样本类型：实时、向后回答、向前/proactive。
3. 解析多选题选项和答案字母，过滤无法配对、缺时间、缺答案、格式异常的 QA。
4. `process_streamo.py` 再根据 `source_video_path` 去不同视频源找原视频或已有帧：
   - ActivityNet 视频；
   - LLaVA-Video YouTube 视频；
   - NeXTVideo 视频；
   - YouCook2 已抽帧；
   - SVBench 视频；
   - 找不到源视频的样本不进入帧目录。
5. 对可用源视频统一抽成 `dataset2/Streamo-Instruct-465K/video/<video_id>/`。

当前保留：

```text
dataset2/Streamo-Instruct-465K/video/
```

Streamo 后续的 unable、one、multi 等 RL 标注构造不在本文维护。

## NeXTVideo

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/NeXTVideo/
```

本地记录中的原始输入：

```text
NExTVideo.zip
train.jsonl
val.jsonl
```

处理脚本：

```text
dataset2/trash/dataset_dirs/NeXTVideo/tmp/process_nextvideo.py
```

处理逻辑：

1. 解压 `NExTVideo.zip` 得到原始 mp4。
2. 读取 `train.jsonl` 和 `val.jsonl`。
3. 从 prompt 中解析问题、A-E 选项和标准答案。
4. 对引用到的视频按 1fps 抽帧。
5. 写入统一的 `video/<video_id>/000000.jpg...` 和 `manifest.json`。

当前保留：

```text
dataset2/NeXTVideo/video/
dataset2/NeXTVideo/sft.jsonl
dataset2/NeXTVideo/rl_tmp.jsonl
```

## LLaVA-Video-178K-YouTube

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/LLaVA-Video-178K/
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/raw/LLaVA-Video-178K-YouTube/videos/
```

旧脚本默认使用的 YouTube 子集：

```text
1_2_m_youtube_v0_1
2_3_m_youtube_v0_1
```

处理脚本：

```text
dataset2/trash/dataset_dirs/LLaVA-Video-178K-YouTube/tmp/process_llava_youtube.py
```

处理逻辑分四步：

1. `filter`：只解析 MCQ，多余的开放式 QA 和 caption 不进入这个转换。
2. `unpack`：只从视频 tar 中解出被 MCQ 标注引用到的视频。
3. `extract`：对候选视频按 1fps 抽帧。
4. `finalize`：写统一标注、`video_index` 和 `video/<video_id>/manifest.json`。

当前保留：

```text
dataset2/LLaVA-Video-178K-YouTube/video/
dataset2/LLaVA-Video-178K-YouTube/sft.jsonl
dataset2/LLaVA-Video-178K-YouTube/rl_tmp.jsonl
dataset2/LLaVA-Video-178K-YouTube/sft_extra_2000.jsonl
```

## LLaVA-Video-178K-PerceptionTest

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/LLaVA-Video-178K/
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/raw/LLaVA-Video-178K-PerceptionTest/videos/
```

旧脚本默认使用的 PerceptionTest 子集：

```text
0_30_s_perceptiontest
30_60_s_perceptiontest
```

处理脚本：

```text
dataset2/trash/dataset_dirs/LLaVA-Video-178K-PerceptionTest/tmp/process_llava_perceptiontest.py
```

处理逻辑：

1. 原始标注中，一个视频 conversation 里可能有多个独立 QA turn。
2. 转换时把每个 human/gpt pair 拆成一条 QA 标注。
3. 同一个视频的多个 QA 共享同一个完整视频输入窗口和同一个帧目录。
4. 只解析 MCQ，并按 1fps 抽帧到统一 `video/<video_id>/`。

当前保留：

```text
dataset2/LLaVA-Video-178K-PerceptionTest/video/
dataset2/LLaVA-Video-178K-PerceptionTest/sft.jsonl
dataset2/LLaVA-Video-178K-PerceptionTest/rl_tmp.jsonl
```

## TimeChat-Online-139K

本地上游源目录：

```text
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/TimeChat-Online-139K/
/mmu_mllm_hdd/zhouhanshu/test/exp3/datasets/raw/TimeChat-Online-139K/frames/
```

原始 QA 文件：

```text
time_chat_online_139k_train_flt.jsonl
```

处理脚本：

```text
dataset2/trash/dataset_dirs/TimeChat-Online-139K/tmp/extract_timechat_frames.py
dataset2/trash/dataset_dirs/TimeChat-Online-139K/tmp/process_timechat.py
```

处理逻辑：

1. 先用 `extract_timechat_frames.py` 把 TimeChat 的帧包整理成完整 1fps frame tree。
2. `process_timechat.py` 读取 `time_chat_online_139k_train_flt.jsonl`。
3. 只使用脚本默认列出的来源：`ActivityNet`、`COIN`、`HiREST`、`TVSum`、`Youcook2`。
4. 从 `videos[0]` 中解析输入窗口的 frame id，重构连续 1fps 帧窗口。
5. 过滤缺帧、缺问答、窗口过短或过长的样本。

当前保留：

```text
dataset2/TimeChat-Online-139K/video/
dataset2/TimeChat-Online-139K/sft.jsonl
```
