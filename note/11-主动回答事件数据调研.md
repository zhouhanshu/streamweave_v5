# 主动回答事件数据调研

日期：2026-05-18

本文只讨论一类数据：用户问题或指令先出现，模型必须在后续某个 answer / response event 或时间窗口到来时回答；其他时间应该保持沉默。这个语义和普通 VideoQA、状态更新 QA 不一样。

## 结论

当前最适合做 RL reward 数据的是：

1. **PhoStream forward**：最贴合。原生标注 `timestamp_question` 和 `timestamp_proactive`，官方评测也把早答判错。
2. **OmniMMI proactive_alerting**：贴合“什么时候说”，但没有标准答案文本，只有触发窗口 `[start, end]`。
3. **ProAssist**：可转成 timed assistant event，但它是任务协助 dialogue，不是 QA answer target。
4. **QIVD / Qualcomm Interactive Video Dataset**：可弱转换成 answer event，但官方语义是“答案证据出现时间”，不是原生 proactive wait-until-answer。

暂时不要直接用：

- `Streamo-Instruct-465K` 里的 `update_when_changed` 样本：这是状态更新 / answer state 语义，不是“只有标注时刻可以答”。如果直接按当前 event-only reward 训练，会把很多可答时间误惩罚成 unprompted answer。
- `RTV-Bench` / StreamingBench / SVBench / CogStream 这类普通 timestamp QA：多数是“在某个时间回答当时问题”或“随时间状态变化”，不是显式 speak/silence event。
- **ProReady-QA**：论文语义非常匹配，但目前没有拿到公开原始 annotation 文件；在数据文件公开前只能作为方向参考。
- **VAPDA-127K**：论文是在线异常预测/检测/分析，方向接近，但没有找到公开原始标注文件；不能作为已确认可转数据。

## 目标格式

我们要的数据应该能落到以下信息：

```json
{
  "query_events": [
    {
      "time": 12.0,
      "content": "Wait until ... then answer ...",
      "answer_policy": "answer_event_only",
      "answer_events": [
        {
          "time": 42.0,
          "answer": "...",
          "content": "..."
        }
      ]
    }
  ]
}
```

如果是窗口标注，则保存为：

```json
{
  "answer_events": [
    {
      "time": 124.0,
      "end_time": 150.0,
      "answer": "",
      "content": ""
    }
  ]
}
```

reward 语义：

- `t < answer_event/window`：模型应该 `<answer></answer>`。
- `t in answer_event/window`：模型应该回答或触发提醒。
- `t > answer_event/window`：如果已经错过窗口，不应把迟答当成满分。

## 数据集逐项确认

### 1. PhoStream

确认等级：**原始标注 + 官方评测代码确认**

来源：

- 标注文件：<https://github.com/Lucky-Lance/PhoStream/blob/main/dataset/phostream_annotations.json>
- 评测代码：<https://github.com/Lucky-Lance/PhoStream/blob/main/run_stream_eval.py>

抽样统计：

```text
videos: 578
responses: 5572
time_type:
  forward: 2802
  instant: 1647
  backward: 1123
```

forward 样例字段：

```json
{
  "user_query": "Wait until she explains her plan and tell me, what is she going to do about her missing scrub pants?",
  "timestamp_question": "00:36",
  "timestamp_proactive": "00:42",
  "time_type": "forward",
  "response": "After realizing she only has scrub tops, she explains that she will have to go to the hospital early to get a scrub bottom from the linen room."
}
```

官方转换逻辑会把 forward 拆成两个 event：

- `timestamp_question`：问题事件，只放 question，不放 response。
- `timestamp_proactive`：回答事件，放 response。

官方评测逻辑里：

- 在 `t < answer_time` 回答：`EarlyResponse = 0`
- 到点不回答：`NoResponse = 0`
- 晚于窗口回答：`LateResponse = 0`

因此 PhoStream forward 和我们要的 reward 语义一致。推荐优先使用。

转换建议：

| PhoStream 字段 | 我们的字段 |
|---|---|
| `video_path` | `video` / `frames_dir` |
| `timestamp_question` | `query_events[].time` |
| `user_query` | `query_events[].content` |
| `timestamp_proactive` | `answer_events[].time` |
| `response` | `answer_events[].answer/content` |
| `time_type=forward` | `answer_policy=answer_event_only` |

注意：

- `instant` 和 `backward` 可以转成 `answer_when_asked`，但不是主动等待型。
- 如果只想训练“该不该答/什么时候答”，先只取 `forward`。

完整例子 1：

```json
{
  "video_path": "youtube_dl_reencoded/downloads_day_in_the_life_vlog_20251002_115415/My_First_day_as_a_Doctor_in_a_New_Hospital_Unfiltered_reencoded/output_1.mp4",
  "user_query": "Wait until she explains her plan and tell me, what is she going to do about her missing scrub pants?",
  "timestamp_question": "00:36",
  "timestamp_proactive": "00:42",
  "time_type": "forward",
  "response": "After realizing she only has scrub tops, she explains that she will have to go to the hospital early to get a scrub bottom from the linen room.",
  "capability": "Audio-Visual Association"
}
```

对应我们的转换：

```json
{
  "query_events": [
    {
      "time": 36.0,
      "content": "Wait until she explains her plan and tell me, what is she going to do about her missing scrub pants?",
      "answer_policy": "answer_event_only",
      "answer_type": "text",
      "answer_events": [
        {
          "time": 42.0,
          "answer": "After realizing she only has scrub tops, she explains that she will have to go to the hospital early to get a scrub bottom from the linen room.",
          "content": "After realizing she only has scrub tops, she explains that she will have to go to the hospital early to get a scrub bottom from the linen room."
        }
      ]
    }
  ]
}
```

证明点：36s 只是提出等待型问题；42s 才有 `timestamp_proactive` 和 response，所以 36-42s 应沉默。

完整例子 2：

```json
{
  "video_path": "youtube_dl_reencoded/downloads_day_in_the_life_vlog_20251002_115415/My_First_day_as_a_Doctor_in_a_New_Hospital_Unfiltered_reencoded/output_1.mp4",
  "user_query": "When she shows the food to the camera, let me know what Turkish dish her mother made for her.",
  "timestamp_question": "01:20",
  "timestamp_proactive": "01:26",
  "time_type": "forward",
  "response": "She opens the blue container and explains that her mom made her Sarma, which she describes as Turkish rice wrapped in leaves. She even holds one up to the camera to show what it looks like.",
  "capability": "Entity & Knowledge Retrieval"
}
```

对应我们的转换：

```json
{
  "query_events": [
    {
      "time": 80.0,
      "content": "When she shows the food to the camera, let me know what Turkish dish her mother made for her.",
      "answer_policy": "answer_event_only",
      "answer_type": "text",
      "answer_events": [
        {
          "time": 86.0,
          "answer": "She opens the blue container and explains that her mom made her Sarma, which she describes as Turkish rice wrapped in leaves. She even holds one up to the camera to show what it looks like.",
          "content": "She opens the blue container and explains that her mom made her Sarma, which she describes as Turkish rice wrapped in leaves. She even holds one up to the camera to show what it looks like."
        }
      ]
    }
  ]
}
```

证明点：query 本身写的是 `When ... let me know`，answer target 在 86s，不是 80s 立即回答。

### 2. OmniMMI

确认等级：**原始标注 + README + 官方评测代码确认**

来源：

- 数据集：<https://huggingface.co/datasets/GuanHuankang/OmniMMI>
- 代码：<https://github.com/OmniMMI/OmniMMI>

OmniMMI 有多个子任务，其中只有 `proactive_alerting` 明确适配当前 reward 语义。

#### proactive_alerting

原始字段：

```json
{
  "video": "x4kKIm8WWOA_360.0_510.0.mp4",
  "question": "Signal me when the video shows the woman in a black coat grooming a horse.",
  "answer": [124, 150]
}
```

统计：

```text
rows: 200
fields: video, question, answer
answer shape: all are [start, end]
```

官方 online inference 对 `pa` 的处理方式是：

1. 在视频开始时输入 `sample["question"]`。
2. 流式喂入视频帧。
3. 记录模型主动 response 的 `response_time`。
4. 官方 eval 检查预测时间是否落在 `answer=[gt_s, gt_e]` 内。

所以它可以直接转成：

| OmniMMI PA 字段 | 我们的字段 |
|---|---|
| `video` | `video` / `frames_dir` |
| `question` | `query_events[].content` |
| `answer[0]` | `answer_events[].time` |
| `answer[1]` | `answer_events[].end_time` |

限制：

- 没有标准答案文本，只有“该提醒/该说话”的时间窗口。
- 更适合训练 silence / speak timing，不适合训练复杂文本回答奖励。
- 如果要塞进当前 answer reward，需要把 `answer` 设成一个固定触发词或空文本事件；judge 只评 timing，不评语义。

完整例子 1：

```json
{
  "video": "x4kKIm8WWOA_360.0_510.0.mp4",
  "question": "Signal me when the video shows the woman in a black coat grooming a horse.",
  "answer": [124, 150]
}
```

对应我们的转换：

```json
{
  "query_events": [
    {
      "time": 0.0,
      "content": "Signal me when the video shows the woman in a black coat grooming a horse.",
      "answer_policy": "answer_event_only",
      "answer_type": "alert",
      "answer_events": [
        {
          "time": 124.0,
          "end_time": 150.0,
          "answer": "alert",
          "content": "The woman in a black coat is grooming a horse."
        }
      ]
    }
  ]
}
```

证明点：官方答案不是文字，而是 `[124, 150]`；模型只要在窗口内触发 response 就算 timing hit。

完整例子 2：

```json
{
  "video": "G7oh5A7gGpQ_210.0_360.0.mp4",
  "question": "Identify the moment in the video when a woman is browsing a makeup selection arranged in wooden crates, indicated by her comments about items being hers and mentioning moisturizer.",
  "answer": [38, 62]
}
```

对应我们的转换：

```json
{
  "query_events": [
    {
      "time": 0.0,
      "content": "Identify the moment in the video when a woman is browsing a makeup selection arranged in wooden crates, indicated by her comments about items being hers and mentioning moisturizer.",
      "answer_policy": "answer_event_only",
      "answer_type": "alert",
      "answer_events": [
        {
          "time": 38.0,
          "end_time": 62.0,
          "answer": "alert",
          "content": "The target moment is happening now."
        }
      ]
    }
  ]
}
```

证明点：和例子 1 一样，`answer` 是触发窗口；窗口外 response 会降低 precision / accuracy。

#### proactive_turntaking

原始字段：

```json
{
  "video": "movie101_2.mp4",
  "conversation": [
    {
      "human": "...",
      "assistant": "..."
    }
  ]
}
```

问题：

- 没有时间字段。
- 官方 PT eval 是判断是否应该轮到模型说话，不是标注每个 answer event 的时间。

结论：`proactive_turntaking` **不直接适合**当前数据格式，除非额外生成 turn time。

### 3. ProAssist

确认等级：**公开数据文件抽样确认**

来源：

- 项目页：<https://pro-assist.github.io/>
- 数据仓库：<https://huggingface.co/594zyc/ProAssist-Dataset>
- 论文：<https://aclanthology.org/2025.emnlp-main.605/>

抽样文件：

```text
processed_data/wtag/generated_dialogs.zip
generated_dialogs/train_filtered.json
```

抽样统计：

```text
rows: 40
turns: 59050
assistant turns: 33800
user turns: 25250
all turns have time
```

assistant turn 样例：

```json
{
  "role": "assistant",
  "time": 14.4,
  "content": "Great! Let's get started. To make a mug cake, we'll need to follow a few steps...",
  "labels": "initiative|instruction",
  "progress": "..."
}
```

ProAssist 的语义是“实时任务协助 assistant 在什么时间说什么”，不是普通 QA。它可以转成我们的 event/silence 格式：

| ProAssist 字段 | 我们的字段 |
|---|---|
| user turn `time/content` | `query_events` 或 QA history |
| assistant turn `time` | `answer_events[].time` |
| assistant turn `content` | `answer_events[].answer/content` |
| assistant turn `labels` | reward 分桶或 metadata |

推荐转换方式：

- 每个 assistant turn 是一个 speak event。
- 非 assistant turn 的 step 都应 silence。
- 不要把 `initiative|instruction`、`responsive|feedback`、`responsive|correction` 全部当成“回答问题”；它们是 assistant action。

适合训练：

- 什么时候应该主动说话。
- 说话内容是否贴合当前任务步骤。
- correction / feedback / instruction 这些交互式行为。

不适合直接混入：

- 当前以 QA answer target 为核心的 `grppo_target_answer`，除非扩展 answer type 为 `assistant_action`。

完整例子 1：

```json
{
  "video_uid": "T24",
  "goal": "Make mug cake",
  "turn_window": [
    {
      "role": "user",
      "time": 13.4,
      "content": "I want to make a mug cake.",
      "labels": ""
    },
    {
      "role": "assistant",
      "time": 14.4,
      "content": "Great! Let's get started. To make a mug cake, we'll need to follow a few steps. First, we need to assemble the cake liner. Can you see the parchment paper on your right side?",
      "labels": "initiative|instruction"
    },
    {
      "role": "assistant",
      "time": 26.9,
      "content": "Take one out and place it inside the mug.",
      "labels": "initiative|instruction"
    }
  ]
}
```

对应我们的转换：

```json
{
  "query_events": [
    {
      "time": 13.4,
      "content": "I want to make a mug cake.",
      "answer_policy": "assistant_action_event",
      "answer_type": "assistant_action",
      "answer_events": [
        {
          "time": 14.4,
          "answer": "Great! Let's get started. To make a mug cake, we'll need to follow a few steps. First, we need to assemble the cake liner. Can you see the parchment paper on your right side?",
          "content": "Great! Let's get started. To make a mug cake, we'll need to follow a few steps. First, we need to assemble the cake liner. Can you see the parchment paper on your right side?",
          "label": "initiative|instruction"
        },
        {
          "time": 26.9,
          "answer": "Take one out and place it inside the mug.",
          "content": "Take one out and place it inside the mug.",
          "label": "initiative|instruction"
        }
      ]
    }
  ]
}
```

证明点：assistant turn 有明确时间和内容，非 assistant turn 的 step 可以训练 silence；但事件是“助理动作”，不是 QA 答案。

完整例子 2：


### 4. Qualcomm Interactive Video Dataset / QIVD

确认等级：**官方 annotation zip 确认**

来源：

- HuggingFace mirror：<https://huggingface.co/datasets/Voxel51/qualcomm-interactive-video-dataset>
- 官方 annotation zip：<https://softwarecenter.qualcomm.com/api/download/software/dataset/AIDataset/QIVD/annotations.zip>

官方 `labels.json` 样例：

```json
{
  "video": "00000000.mp4",
  "question": "What am I holding in my left hand?",
  "answer": "You are holding a Rubik's cube in your left hand.",
  "short_answer": "A Rubik's cube",
  "timestamp": "00:04.4",
  "category": "object referencing",
  "id": 1972
}
```

统计：

```text
labels: 2900
fields: video, question, answer, short_answer, timestamp, category, id
```

这个数据可以转，但语义弱于 PhoStream：

- `timestamp` 是“答案证据能在视频中找到的时间”。
- 官方没有 `timestamp_question`。
- 官方没有显式要求模型等待，也没有显式 silence 标注。

可行转换：

| QIVD 字段 | 我们的字段 |
|---|---|
| `video` | `video` / `frames_dir` |
| `question` | `query_events[].content` |
| 人工设定 `query_time=0` 或 `max(0, timestamp - pre_wait)` | `query_events[].time` |
| `timestamp` | `answer_events[].time` |
| `answer` / `short_answer` | `answer_events[].answer/content` |

风险：

- 如果问题本身不是 “wait until ...”，模型在开头被问后不答是人为构造的。
- 对一些简单视觉问题，答案可能在 `timestamp` 之前已经可见；用单点 timestamp 当唯一可答时间会引入噪声。

建议：

- QIVD 可以作为补充数据，但要加 prompt 模板：`Wait until the answer is visually supported, then answer...`
- 训练时最好给一个窗口，比如 `[timestamp - 1s, timestamp + 2s]`，不要硬单点。
- 语义优先级低于 PhoStream forward 和 OmniMMI PA。

完整例子 1：

```json
{
  "video": "00000000.mp4",
  "question": "What am I holding in my left hand?",
  "answer": "You are holding a Rubik's cube in your left hand.",
  "short_answer": "A Rubik's cube",
  "timestamp": "00:04.4",
  "category": "object referencing",
  "id": 1972
}
```

弱转换：

```json
{
  "query_events": [
    {
      "time": 0.0,
      "content": "Wait until the answer is visually supported, then answer: What am I holding in my left hand?",
      "answer_policy": "answer_event_only",
      "answer_type": "text",
      "answer_events": [
        {
          "time": 4.4,
          "answer": "A Rubik's cube",
          "content": "A Rubik's cube"
        }
      ]
    }
  ]
}
```

证明点：它确实有 answer timestamp 和答案文本；但 `timestamp` 是证据时间，不是官方 proactive response time。

完整例子 2：

```json
{
  "video": "00000001.mp4",
  "question": "Am I standing sideways?",
  "answer": "Yes, you are standing sideways.",
  "short_answer": "Yes",
  "timestamp": "00:05.2",
  "category": "action attributes",
  "id": 9316
}
```

弱转换：

```json
{
  "query_events": [
    {
      "time": 0.0,
      "content": "Wait until the answer is visually supported, then answer: Am I standing sideways?",
      "answer_policy": "answer_event_only",
      "answer_type": "text",
      "answer_events": [
        {
          "time": 5.2,
          "answer": "Yes",
          "content": "Yes"
        }
      ]
    }
  ]
}
```

证明点：可转成 answer event，但这类 yes/no 问题可能在 timestamp 前已经可见，所以要加 answer window，不能硬把 5.2s 当唯一正确瞬间。

## 其他数据集状态

### ProReady-QA / StreamReady

确认等级：**论文语义确认，原始标注未确认**

来源：

- arXiv：<https://arxiv.org/abs/2603.08620>
- Microsoft Research 页面：<https://www.microsoft.com/en-us/research/publication/streamready-learning-what-to-answer-and-when-in-long-streaming-videos/>

论文摘要和项目说明都明确写到：

- models need to answer exactly when supporting visual evidence appears
- answering before evidence is speculation
- ProReady-QA has annotated answer evidence windows
- proactive multi-turn questions

这说明它的语义非常贴合我们要的数据。但当前没有找到公开原始 annotation JSON / HF dataset / GitHub data 文件。因此：

- 可以作为目标数据形态参考。
- 暂时不能作为可直接转换的数据源。

### VAPDA-127K / AssistPDA

确认等级：**论文方向接近，原始标注未确认**

来源：

- arXiv：<https://arxiv.org/abs/2503.21904>

论文说它是 online video anomaly prediction, detection, and analysis，包含 event-level anomaly prediction，可以在异常完全展开前 proactive forecasting。方向接近“事件触发说话/预警”，但没有找到公开数据标注文件。

当前不能确认：

- 是否有明确 answer/alert timestamp。
- 是否有 silence / no-response 标签。
- 是否有 answer text 或 anomaly explanation target。

结论：暂不纳入训练数据候选，等代码和数据公开后再看。

```text

answer_events: 2931

answer_policy/type:
  answer_when_asked + mcq: 779
  answer_when_asked + text: 1099
  update_when_changed + mcq: 1002

update_when_changed:
  1 answer event: 954
  2 answer events: 45
  3 answer events: 3
  source: Streamo-Instruct-465K
```

这里的问题是 `update_when_changed`。它更像“状态变化时答案更新”，不等价于“只有标注时刻可以说话”。如果 reward 按 event-only 解释，就会误惩罚一些合理回答：

- 标注时刻之后，状态仍然成立，模型回答可能合理。
- 但当前 event-only reward 会认为“没有 current answer target，所以不该答”。

这和 PhoStream forward 完全不同。PhoStream forward 的 `timestamp_proactive` 是“现在才该答”，早答就是猜；`update_when_changed` 则往往是“答案状态从这里开始变成某个值”。

## 推荐路线

短期最稳：

1. 新建一个 proactive-event 数据池，只放：
   - PhoStream `time_type=forward`
   - OmniMMI `proactive_alerting`
2. 这个池统一设置：
   - `answer_policy=answer_event_only`
   - 非 answer window 强 silence reward
3. 暂时从当前 RL 主池里剥离 `update_when_changed`，或者单独给它做 persistent-state reward。

中期：

1. 把 ProAssist 转成 `assistant_action` 类型，不和 QA answer target 混。
2. 把 QIVD 转成弱 proactive QA，并加宽 answer window。
3. 等 ProReady-QA 原始标注公开后，优先接入。

不建议：

- 不要把所有带 timestamp 的 VideoQA 都当成 proactive answer event。
- 不要让规则 answer target 强压 LLM judge：如果是开放文本答案，语义等价应由 judge 或归一化匹配处理。
- 不要把 `update_when_changed` 继续按 event-only silence reward 训练。
