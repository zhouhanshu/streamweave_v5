### 打分流程设计

把原视频和模型压缩后的 memory 都变成一组带时间的 embedding units：

```
video_units:
  v_i = 原视频第 i 个语义单元（1fps frame、scene keyframe 或 2-4s clip）

memory_units:
  m_j = note 图像 embedding 或 bridge 文本 embedding
  每个 m_j 都带 t_start/t_end
```

计算时间约束下的相似度矩阵（**必须加 temporal mask，否则泛泛的 bridge 会匹配整段视频导致 reward 虚高**）：

```
S[i, j] = cosine(embed(video_unit_i), embed(memory_unit_j))
allow(i, j) = overlap(video_t_i, memory_t_j) 或距离小于 delta
S[i, j] = -inf if not allow(i, j)
```

---

### 推荐公式

第一版用 greedy matching（类 BERTScore F1）：

```
coverage_i  = max_j S[i, j]
precision_j = max_i S[i, j]

coverage  = mean_i coverage_i
precision = mean_j precision_j

R_semantic = 2 * coverage * precision / (coverage + precision + eps)
```

cosine 需先校准到 [0, 1]，否则负数会让 F1 不稳定：

```
sim01 = clamp((cosine - tau_neg) / (tau_pos - tau_neg), 0, 1)
```

`tau_neg` / `tau_pos` 用验证集估计：
- `tau_neg`：随机不同视频之间的 image-text / image-image 相似度均值或 P95
- `tau_pos`：同视频附近帧、同时间 caption 的相似度均值或 P50

---

### Step-level + Trajectory-level 联合

```
R_sem_step = F1(current_window_video_units, current_step_note_bridge_units)
R_sem_traj = F1(full_video_units, all_memory_units)
R_semantic  = 0.5 * mean(R_sem_step) + 0.5 * R_sem_traj
```

---

### 成本和冗余惩罚（必须加，否则模型会狂存帧或写长 bridge）

```
R_compress =
    R_semantic
  - lambda_img * num_notes
  - lambda_txt * bridge_tokens / 32
  - lambda_red * redundancy

redundancy = mean_{j<k} max(0, sim(memory_j, memory_k) - 0.85)
```
