## 过程指标对比

统计口径：

- note/frame 占比：`applied notes / trace 中唯一 current frame 图片数`。
- note 间隔：同一样本内相邻 note 起始时间差。
- parser bad：`quality.parser_ok=False`。
- format bad：`format_reward=0`。
- raw invalid：`quality.valid=False`，包含时间 gap、open-tail、event overlap 等结构问题，不等同于 XML parser 错误。

- `Gemini no-state / eta` 的 note 最稀疏，note/frame 只有 `4.78%`，平均 note 间隔 `20.95s`，但 raw invalid 很高，主要是 event overlap、bridge time 越界和 open-tail 问题。

## Backward 退化分析

Qwen3-VL answered-full SFT 的 Backward AVG 比 base 低：`58.46 -> 55.37`。这不是三个 Backward task 全面退化：

| Task | Base | SFT | 变化 |
| --- | ---: | ---: | ---: |
| EPM | 51.35 | 59.46 | +8.11 |
| ASI | 63.16 | 63.16 | 0.00 |
| HLD | 60.87 | 43.48 | -17.39 |

逐题对比显示，base 对但 SFT 错有 `8` 题，SFT 对但 base 错有 `7` 题，净差只有 `1` 题；但 OVO 是 task-macro，HLD 少对 `4/23` 会明显拉低 Backward AVG。

核心原因是 HLD 的 `Unable to answer` 校准变差。当前 1/8 切片里的 HLD GT 都对应某个 `Unable to answer` 选项，SFT 更容易根据 Memory 或当前帧硬猜最像的具体选项，而不是在证据不精确、选项不匹配时选择 unable。

典型例子：

- `390`：问题问 “What did I put in bottle on sink?”，GT 是 `Unable to answer`。SFT state 里写了没有看到 sink，却把早期 motorcycle oil 记忆迁移过来，答 `A oil`。
- `422`：问题问 living room 里哪里能坐，视频实际是 bedroom。SFT 看到 bedroom 里有 chair，答 `A On the chair`，忽略了 location mismatch。
- `462`：问题问 chandelier hanging over what，GT 是 `Unable to answer`。SFT 把卧室里的灯具当 chandelier，并映射到 `E living room`。
- `309`：问题问 wire 被拿起前在哪里，SFT 判断像是在 workbench，但选项没有 workbench，于是硬映射到 `B at the box`。

EPM 里 SFT 总体提升，但也出现相反问题：例如 `260` 没有 Unable 选项，问题问 blue pillow pillowcase 材质。base 猜 `B Velvet` 正确，SFT 认为证据不足而保持空答，导致错误。这说明 SFT 的 abstention 边界还不稳定：HLD 该 unable 时会硬猜，普通选择题该根据选项给最佳答案时又可能沉默。

结论：SFT 学到了更稳定的 XML 格式和更积极的 answer 更新，因此 Forward 明显提升；但它没有学好 HLD 这类 “问题指向的对象/地点/状态必须精确存在，否则选择 Unable” 的判定边界。

如果不看 HLD，SFT 相比 base 是明确提升的：

| 指标 | Base | SFT | SFT - Base |
| --- | ---: | ---: | ---: |
| Backward 不含 HLD，只算 EPM+ASI | 57.25 | 61.31 | +4.05 |
| Realtime | 77.54 | 79.49 | +1.95 |
| Forward | 37.68 | 48.99 | +11.31 |
| Total，Backward 按不含 HLD 重算 | 57.49 | 63.26 | +5.77 |

因此当前实验可以解读为：SFT 的总体收益存在，尤其 Forward 提升明显；Backward 的负面主要集中在 HLD 的 unable-answer 判断边界，而不是新协议整体无效。
