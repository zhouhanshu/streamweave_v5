- `full` 和 `1/8` 不能直接横比，`1/8` 只做 sanity check。

## Note 与 Memory 压缩统计

### 计算口径

Qwen3VL 本地配置：

```text
patch_size=16
spatial_merge_size=2
temporal_patch_size=2
```

当前评测把视频帧作为独立图片喂入，不按 video tensor 合并，所以每帧视觉 token 估算为：

```text
visual_tokens_per_frame = (resolution / (patch_size * spatial_merge_size))^2
448 -> (448 / 32)^2 = 196
768 -> (768 / 32)^2 = 576
```

统计字段说明：

- `Eval frames` 用 `num_steps * frames_per_step` 估算；当前相关正式结果都是 `frames_per_step=5`，最后一个 step 可能不足 5 帧，所以这是略偏大的分母。
- `Note/frame` 是生成的 note 帧数 / `Eval frames`。
- `Final mem/video tok` 是 final memory 的 token 量 / 全部视频帧视觉 token 量，计算为 `memory_text_tokens + active_image_notes * visual_tokens_per_frame`。其中 memory 文本用本地 Qwen tokenizer 计数，图片路径替换成 `<image>`，`image_evicted` 的 note 不算视觉 token。
- `Upper mem/video tok` 是上界，把所有生成过的 note 都当作仍保留图片 token。

## 评测注意事项

- step200 SFT 导出模型路径曾为 `models/qwen3vl8b_sft_anchor_delta_step200_vllm`。
- 这是 full fine-tuned HF 模型，不是 LoRA adapter。
- LLaMAFactory export 后可能在 `tokenizer_config.json` 写入 list 形式的 `extra_special_tokens`，本地 vLLM/Transformers 会报 `list has no attribute keys`；需要删除该字段，并从 base model 补齐 tokenizer/processor sidecar。
- 8 卡 OVO vLLM 脚本会启动 `127.0.0.1:8000-8007`，输出包括 `results.jsonl`、`results_summary.json/md`、`traces/`、`worker_logs/`、`vllm_logs/`。
- 当前可见输出协议应是 `<state>/<answer>/<anchor>/<delta>`；内部旧名 `NoteRecord/BridgeRecord/event.kind=="note"` 只是兼容历史代码。
