### Exp #1 · VideoHallucer × anchor_delta × qwen3vl_sft_0516_step50

- **日期**:2026-05-16 23:42 启动,2026-05-17 完成
- **状态**:✅ 完成
- **模型**:`/mmu_mllm_hdd/zhouhanshu/test/exp3/streamweave_v5/models/qwen3vl_sft_0516_step50`
  - 17 GB 单 safetensors,Qwen3-VL-8B-Instruct 基底 + StreamWeave SFT(step 50)
- **Benchmark / Policy**:VideoHallucer × `anchor_delta`(图文交错 memory)
- **Run dir**:`dataset3/outputs/vh_anchor_delta_qwen3vl_sft_0516_step50_20260516_234223/`
- **命令**:
  ```bash
  ./dataset3/scripts/run_videohallucer_8gpu.sh \
    /mmu_mllm_hdd/zhouhanshu/test/exp3/streamweave_v5/models/qwen3vl_sft_0516_step50 \
    anchor_delta
  ```
- **关键超参**:`frames_per_step=5`,`sample_fps=1.0`,`resolution=768`,`temperature=0`,`prompt=teacher_eval`,8 GPU × 16 workers
- **结果**:

  ```
  Subset                   pairs     basic    halluc   overall
  ────────────────────────────────────────────────────────────
  external_factual           200    0.7850    0.5950    0.4350
  external_nonfactual        200    0.7900    0.8150    0.6150
  fact_detect                 50    0.9800    0.6400    0.6200
  interaction                124    0.1290    0.7742    0.0726
  object_relation            200    0.3500    0.3350    0.2450
  semantic_detail            200    0.5100    0.5250    0.2750
  temporal                   176    0.5057    0.7273    0.3693
  ────────────────────────────────────────────────────────────
  OVERALL (macro)           1150    0.5785    0.6302    0.3760
  ```

- **观察**:
  1. **Overall 37.60** — 在 7B/8B 段位接近 PLLaVA-7B(38.1),高于 LLaVA-NeXT-Video-DPO-7B(32.0),高于 Gemini-1.5-Pro 闭源(37.8)
  2. **`interaction` 暴跌(basic=0.13,overall=0.07)**:这个 subset 是改否定句式翻转答案("Doesn't the man...?" yes / "Does the man...?" no),basic 单题已经塌了。**模型对否定句式不敏感**,不算反幻觉问题,**两个 policy 都会差**(不影响 anchor vs delta 对照)
  3. **`fact_detect` 最高(0.62)**:长文本一致性,SFT 模型保留了语言能力
  4. **`external_nonfactual` 第二(0.615)**:反常识题靠先验也能答,**预测 delta_only 跟 anchor_delta 差异最小**
  5. **`object_relation` 最低(0.245)**:物体存在性识别弱
  6. **`semantic_detail` 0.275**:双视频细粒度区分,**最关键的对照 subset**,等 delta_only 数字
  7. 答非 yes/no 比例低(`no_match` 看 per_qa.jsonl 估算),模型遵守了 "Yes/No only" 指令
- **跟 paper Table 3 / 4 同档位模型对照**(均为 7B):

  | 模型 | Overall | ORH | SDH |
  |---|---|---|---|
  | PLLaVA-7B | 38.1 | 60.0 | 57.0 |
  | **本 run(anchor_delta)** | **37.60** | **24.5** | **27.5** |
  | LLaVA-NeXT-Video-DPO-7B | 32.0 | 51.5 | 38.0 |
  | LLaMA-VID-7B | 21.0 | 43.5 | 17.0 |
  | MiniGPT4-Video | 22.3 | 27.5 | 23.5 |

  - Overall 总分跟 PLLaVA 平,**但单看 ORH/SDH 反而落后** — 说明强项在 external/fact 那 3 个 subset
  - 反幻觉能力在不依赖视频的 subset(`external_nonfactual` / `fact_detect`)上突出,**纯视觉 subset(物体/细节)上偏弱**
- **下一步**:
  - 等 **Exp #2:同模型 × `delta_only`** 跑完 → 出对照表
  - 重点看 `semantic_detail` / `object_relation` / `temporal` 三个**视觉依赖强**的 subset 差值是否显著
  - 如果差值显著(图文 > 纯文本 ≥ 3pp),命题在 SFT 模型上立得住

## 已知风险 / 注意事项

- **GRPO 0509 模型的 vLLM 兼容性未验证**:跑 Exp #5/#6 前必须先 dry-run 一次 vLLM 加载,避免 8 卡全开后才发现挂不起来
- **`mix` 视频目录映射**:eventhallusion.json 里 `source_id=mix_001` 对应 `video_id=interleave_001`,convert 已自动处理,但日后做 GPT judge desc 任务时要按 source_id 输出
- **scorer 跟官方对齐**:VH 用 lenient `\b{gt}\b`(模型说"No, the answer is yes"两边都 hit),EH 用 strict `startswith`(`"  Yes."`不匹配)。这是官方行为,不是 bug
- **2 份 config 严格对照**:除 `policy:` 外不要乱改;改其他超参就必须重跑两边
- **磁盘**:`/mmu_mllm_hdd/` 当前使用率 100%(可用 17T),每个 run trace 可能几 GB,定期清旧 trace
